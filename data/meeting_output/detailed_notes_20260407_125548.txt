**📘 1. Database Types**

📚 **What is a Database?**
A database is a collection of organized data that can be easily accessed, managed, and updated.

📝 **Structured vs Unstructured Data**

🔹 **Structured Data**
✅ Organized and follows a specific format
✅ Easy to search, retrieve, and manipulate
✅ Examples: relational databases (e.g., MySQL), spreadsheets (e.g., Excel)

```json
{
  "name": "John Doe",
  "age": 30,
  "address": {
    "street": "123 Main St",
    "city": "Anytown",
    "state": "CA",
    "zip": "12345"
  }
}
```

🔹 **Unstructured Data**
✅ Does not follow a specific format
✅ Difficult to search, retrieve, and manipulate
✅ Examples: text documents, images, videos, audio files

**Comparison Table: Structured vs Unstructured Data**

|  | Structured Data | Unstructured Data |
| --- | --- | --- |
| **Format** | Follows a specific format | Does not follow a specific format |
| **Searchability** | Easy to search | Difficult to search |
| **Manipulation** | Easy to manipulate | Difficult to manipulate |

**📘 2. NoSQL Databases**

📚 **What is a NoSQL Database?**
A NoSQL database is a type of database that does not use the traditional table-based relational model.

🔹 **Not Only SQL (NoSQL)**
✅ Stores data in a different way than traditional relational databases
✅ Examples: MongoDB, Cassandra, Redis

📝 **MongoDB: An Example of a NoSQL Database**
MongoDB is a popular NoSQL database that stores data in a JSON-like format.

```json
{
  "_id": ObjectId,
  "name": "John Doe",
  "age": 30,
  "address": {
    "street": "123 Main St",
    "city": "Anytown",
    "state": "CA",
    "zip": "12345"
  }
}
```
