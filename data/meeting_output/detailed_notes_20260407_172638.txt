**📘 1. Database Overview**

📚 **What is a Database?**
A database is a collection of organized data that is stored in a way that allows for efficient retrieval and manipulation.

🔹 **Key Features:**
✅ Data is stored in a structured format
✅ Data can be queried and manipulated using various operations
✅ Data is typically stored on a physical device, such as a hard drive or solid-state drive

📊 **Types of Databases:**
- Relational databases (e.g., MySQL, PostgreSQL)
- NoSQL databases (e.g., MongoDB, Cassandra)

**📚 MongoDB:**
MongoDB is a type of NoSQL database that stores data in a JSON-like format.

🔹 **Key Features:**
✅ Data is stored in a flexible, JSON-like format
✅ Data can be queried using a variety of operations, including filtering and aggregation
✅ MongoDB is designed for high scalability and performance

**📊 Unstructured vs Structured Data:**
Unstructured data refers to data that doesn't follow a specific format, such as images, videos, and audio files. Structured data, on the other hand, refers to data that is organized into tables, spreadsheets, and other formats.

🔹 **Key Features:**
✅ Unstructured data doesn't follow a specific format
✅ Structured data is organized into tables and spreadsheets
✅ Key difference: unstructured data doesn't follow a specific format

**📚 Example:**
```json
// Unstructured data (image)
{
  "image": "data:image/jpeg;base64,/9j/4AAQSkZJRgABAQEASABIAAD/..."
}

// Structured data (table)
{
  "name": "John Doe",
  "age": 30,
  "city": "New York"
}
```

**📊 Comparison Table:**

|  | Unstructured Data | Structured Data |
| --- | --- | --- |
| **Format** | No specific format | Organized into tables and spreadsheets |
| **Querying** | Difficult to query | Easy to query using various operations |
| **Storage** | Can be stored in various formats | Typically stored in a database |

**📊 2. NoSQL vs SQL**

📚 **What is NoSQL?**
NoSQL is a type of database that doesn't follow traditional table-based structure.

🔹 **Key Features:**
✅ Doesn't follow traditional table-based structure
✅ Data is stored in a flexible, JSON-like format
✅ NoSQL is designed for high scalability and performance

📚 **What is SQL?**
SQL is a traditional database management system that uses a structured query language to manage data.

🔹 **Key Features:**
✅ Follows traditional table-based structure
✅ Data is stored in a structured format
✅ SQL is designed for transactional data

**📊 Comparison Table:**

|  | NoSQL | SQL |
| --- | --- | --- |
| **Structure** | Flexible, JSON-like format | Traditional table-based structure |
| **Querying** | Easy to query using various operations | Easy to query using SQL |
| **Scalability** | High scalability and performance | Limited scalability |

**📚 Example:**
```json
// NoSQL (MongoDB)
{
  "name": "John Doe",
  "age": 30,
  "city": "New York"
}

// SQL (relational database)
CREATE TABLE users (
  id INT PRIMARY KEY,
  name VARCHAR(255),
  age INT,
  city VARCHAR(255)
);
```
